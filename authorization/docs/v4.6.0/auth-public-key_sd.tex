\section{Overview}
\label{sec:overview}
This document describes the \textbf{auth-public-key} service, which provides the public key of the system. It is necessary for validating an access token generated by the Authorization Core System.

The rest of this document is organized as follows.
In Section \ref{sec:functions}, we describe the abstract message functions provided by the service.
In Section \ref{sec:model}, we end the document by presenting the data types used by the mentioned functions.

\subsection{How This Service Is Meant to Be Used}
The provider systems are recommended to use this service at their start up in order to already have this key when a service consumption request arrives and the access the token need to be validated.

\subsection{Important Delimitations}
\label{sec:delimitations}

The public key string must be a Base64 encoded JSON string.

\subsection{Access policy}
\label{sec:accesspolicy}

Available for anyone within the local cloud.
